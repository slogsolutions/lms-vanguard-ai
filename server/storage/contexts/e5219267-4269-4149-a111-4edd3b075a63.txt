UNIT I: INTRODUCTION & NUMBER THEORY  1. Network Security & OSI Model  Network security refers to the practices and technologies used to protect data, devices, and networks from cyber threats. It ensures that only authorized users can access network resources while preventing hackers, malware, and unauthorized access. The   OSI (Open Systems Interconnection) Model   is a framework that defines how different networking devices communicate over a network. It has   seven layers , each with specific functions:  1.   Physical Layer   – Deals with hardware (cables, switches). 2.   Data Link Layer   – Handles MAC addresses and error detection. 3.   Network Layer   – Manages IP addresses and routing. 4.   Transport Layer   – Ensures reliable data transmission (TCP, UDP). 5.   Session Layer   – Manages and maintains communication sessions. 6.   Presentation Layer   – Encrypts and formats data. 7.   Application Layer   – Provides services like HTTP, FTP, and email.  2. Services, Mechanisms, and Attacks  Network security involves   services, mechanisms, and attacks :     Security Services : Protect data and communication, ensuring confidentiality, integrity, authentication, and access control.     Security Mechanisms : Techniques like encryption, firewalls, and authentication protocols used to enforce security services.     Security Attacks : Attempts by hackers to steal, modify, or destroy data.  3. OSI Security Architecture  The   OSI Security Architecture   defines security services and mechanisms applied at different layers of the OSI model to protect communication. It covers:     Authentication : Verifies the identity of users.     Access Control : Restricts unauthorized access.     Data Integrity : Ensures data is not altered.     Confidentiality : Protects data from being accessed by unauthorized parties.     Non-repudiation : Ensures that the sender cannot deny sending a message.  4. Network Security Model
A   Network Security Model   represents how different security components work together to protect a network. It typically consists of:  1.   Sender   – The user or device sending data. 2.   Receiver   – The user or device receiving data. 3.   Security Policies   – Rules and guidelines for secure communication. 4.   Security Mechanisms   – Tools like firewalls, antivirus, encryption, and authentication. 5.   Attackers   – Hackers trying to exploit network vulnerabilities. 6.   Network Infrastructure   – Routers, servers, and communication channels.  Classical Encryption Techniques  Classical encryption techniques are the early methods used to secure messages by transforming plain text into an unreadable format. These methods were mainly used before the development of modern cryptography.  1. Symmetric Cipher Model  The   Symmetric Cipher Model   is an encryption method where the same key is used for both   encryption   and   decryption . It is called   symmetric   because both the sender and receiver share the same secret key.  How it Works:  1.   The sender encrypts the plaintext using a secret key. 2.   The receiver decrypts the ciphertext using the same secret key.  ✅   Advantages:     Fast and efficient.     Suitable for large amounts of data.  ❌   Disadvantages:     If the key is stolen or shared with an attacker, the encryption becomes useless.     Key distribution is a challenge.  Example:   AES, DES, and the Caesar cipher.  2. Substitution Techniques  In   substitution techniques , each letter in the plaintext is replaced with another letter or symbol.  Types of Substitution Techniques:
Caesar Cipher   – Each letter is shifted by a fixed number (e.g., shifting A to D, B to E, etc.).  o   Example:     Plaintext:   HELLO     Shift by 3:   KHOOR  Monoalphabetic Cipher   – Each letter is replaced with a randomly assigned letter.  o   Example:     Plaintext:   HELLO     Ciphertext:   XQCC  Playfair Cipher   – Uses a 5×5 grid of letters to replace digraphs (pairs of letters).  Vigenère Cipher   – Uses multiple Caesar ciphers based on a keyword.  ✅   Advantages:     Simple to use.     Easy to implement.  ❌   Disadvantages:     Can be broken using frequency analysis.     Not secure for modern applications.  3. Transposition Techniques  In   transposition techniques , the positions of letters are changed instead of replacing them.  Types of Transposition Techniques: Rail Fence Cipher   – The message is written in a zigzag pattern and then read row by row.  Columnar Transposition Cipher   – The message is written in rows, and then the columns are rearranged based on a key.  ✅   Advantages:     Harder to break than substitution.     Can be combined with substitution for stronger encryption.  ❌   Disadvantages:
   Still vulnerable to pattern recognition.  4. Steganography  Steganography   is the technique of hiding a message within another object (such as an image, audio, or text) so that its presence is not detected.  Common Methods of Steganography:     Text Steganography   – Hiding messages within spaces or invisible characters.     Image Steganography   – Hiding data within image pixels.     Audio Steganography   – Embedding messages in audio files.  ✅   Advantages:     The message remains hidden, unlike encryption, which just scrambles data.  ❌   Disadvantages:     If detected, the hidden message can be removed.     Not as secure as encryption for critical data.  Conclusion  Classical encryption techniques like   substitution, transposition, and steganography  were effective in ancient times but are no longer secure against modern attacks. Today, they are mainly used for learning purposes or combined with modern cryptographic techniques for added security.  Finite Fields and Number Theory  Number theory and finite fields are fundamental to cryptography and computer science. These concepts help in encryption, error detection, and solving mathematical problems efficiently.  1. Groups, Rings, and Fields  These are algebraic structures that define how numbers interact with operations like addition and multiplication.  Group     A set of elements with an operation (like addition or multiplication) that satisfies:
1.   Closure   – Operation on two elements results in another element in the set. 2.   Associativity   – (a ∘   b) ∘   c = a ∘   (b ∘   c). 3.   Identity   – There is an element "e" such that a ∘   e = a. 4.   Inverse   – Each element has an inverse (b such that a ∘   b = e).     Example:   Integers under addition (Z, +).  Ring     A set with   two   operations (addition and multiplication), satisfying group properties for addition and some multiplication rules.     Example:   Integers (Z, +, ×).  Field     A ring where every nonzero element has a multiplicative inverse (a × a⁻¹ = 1).     Example:   Real numbers (R, +, ×) or finite fields (GF(p)) used in cryptography.  2. Modular Arithmetic  Modular arithmetic works with numbers in a   fixed range (mod n) . It is essential in cryptography.  Basic Rule:  a≡bmod na \equiv b \mod na≡bmodn This means when   a   is divided by   n , the remainder is   b .  Examples:     Addition:   (7 + 5) mod 4 = 12 mod 4 =   0     Multiplication:   (3 × 4) mod 5 = 12 mod 5 =   2     Exponentiation:   (3⁴) mod 5 = 81 mod 5 =   1  Modular arithmetic is used in encryption algorithms like RSA.  3. Euclid’s Algorithm  Used to find the   Greatest Common Divisor (GCD)   of two numbers.  Algorithm:  1.   If   b = 0 , then GCD(a, b) =   a . 2.   Otherwise, compute GCD(b, a mod b) recursively.  Example:   Find GCD(48, 18)
   48 mod 18 = 12     18 mod 12 = 6     12 mod 6 = 0     GCD = 6  This is used in cryptographic key generation.  4. Finite Fields  A   finite field   (also called   Galois Field, GF(p) ) is a field with a finite number of elements.  Properties:  1.   Contains a fixed number of elements   p   or   pⁿ   (where p is prime). 2.   All operations (addition, multiplication) are done modulo p.  Example:   GF(5) has elements {0, 1, 2, 3, 4}.     Addition: (3 + 4) mod 5 =   2     Multiplication: (2 × 4) mod 5 =   3  Finite fields are used in AES encryption and ECC (Elliptic Curve Cryptography).  5. Polynomial Arithmetic  Polynomial arithmetic is similar to regular arithmetic but performed with polynomials.  Operations:     Addition: (x² + 2x + 1) + (x³ + x) =   x³ + x² + 3x + 1     Multiplication: (x + 1) × (x² + x) =   x³ + 2x² + x  Polynomials are used in   error correction (Reed-Solomon codes)   and   cryptographic functions .  6. Prime Numbers  A   prime number   is a number that has only two factors:   1 and itself .  Examples:   2, 3, 5, 7, 11, 13, 17, 19, ...
Prime numbers are essential in cryptography because they form the basis of   public- key encryption (RSA) .  7. Fermat’s and Euler’s Theorems  These theorems help in computing modular inverses and exponents efficiently.  Fermat’s Theorem:  If   p   is a prime and   a   is not divisible by   p , then:  ap − 1≡1mod   pa^{p-1} \equiv 1 \mod pap − 1≡1modp  Euler’s Theorem:  For any number   a   and   n   (where GCD(a, n) = 1):  aφ(n)≡1mod   na^{φ(n)} \equiv 1 \mod naφ(n)≡1modn  where   φ(n)   is the   Euler totient function   (count of numbers ≤ n that are coprime to n).  Example:   φ(9) = 6 because numbers coprime to 9 are {1, 2, 4, 5, 7, 8}. These theorems are used in RSA encryption.  8. Testing for Primality  To check if a number   n   is prime, we use:  Basic method:     Check divisibility by numbers up to   √n .  Miller-Rabin Test (Probabilistic method):     Tests whether   n   behaves like a prime using random numbers.  AKS Primality Test (Deterministic method):     A complex test that runs in polynomial time.  Primality testing is used in key generation for cryptographic systems.  9. Chinese Remainder Theorem (CRT)
CRT helps solve simultaneous modular equations.  Example:  Find   x   that satisfies:  x≡2mod   3x \equiv 2 \mod 3x≡2mod3 x≡3mod   5x \equiv 3 \mod 5x≡3mod5 x≡2mod   7x \equiv 2 \mod 7x≡2mod7  Using CRT, we find   x = 23 . CRT is useful in   RSA encryption   to speed up calculations.  10. Discrete Logarithms  A   discrete logarithm   is the inverse of exponentiation in modular arithmetic.  Definition:  If  bx≡ymod   pb^x \equiv y \mod pbx≡ymodp  then   x   is the discrete logarithm of   y   to the base   b .  Example:  Find   x   in:  3x≡4mod   73^x \equiv 4 \mod 73x≡4mod7  By trying values:     31mod 7=33^1 \mod 7 = 331mod7=3     32mod 7=23^2 \mod 7 = 232mod7=2     33mod 7=63^3 \mod 7 = 633mod7=6     34mod 7=43^4 \mod 7 = 434mod7=4   ✅  So,   x = 4 .  Use in Cryptography:  Discrete logarithms are the basis of   Diffie-Hellman key exchange   and   Elliptic Curve Cryptography (ECC) .  Conclusion  These number theory concepts are the foundation of modern cryptography. They help create secure encryption systems, digital signatures, and authentication mechanisms.
Understanding them is key to designing secure systems in cybersecurity and cryptograph  UNIT II: BLOCK CIPHERS & PUBLIC KEY CRYPTOGRAPHY Block Ciphers and Encryption Algorithms  Block ciphers are encryption techniques that process fixed-size blocks of plaintext at a time, transforming them into ciphertext using a secret key. They are widely used in cryptography to secure data.  1. Data Encryption Standard (DES)  DES is one of the earliest and most famous encryption algorithms, developed by IBM in the 1970s and later adopted by the U.S. government.  Key Features of DES:     Block Size:   64 bits (processes 64-bit blocks at a time).     Key Length:   56 bits (actually a 64-bit key, but 8 bits are used for parity checking).     Rounds:   16 rounds of complex encryption operations.     Structure:   Follows a   Feistel Network , which divides the data into two halves and applies multiple encryption functions.  Weaknesses:     DES is vulnerable to brute force attacks because of its short key length (56 bits).     It is now considered   insecure   for modern applications.  ✅   Used in:   Older banking systems and legacy encryption.  2. Block Cipher Principles  Block ciphers use a fixed-length block of plaintext and apply encryption in multiple rounds using substitution and permutation.  Key Concepts in Block Cipher Design:  1.   Confusion:   Makes the relationship between the plaintext, key, and ciphertext complex.  o   Achieved using   substitution   (replacing bits with new values). 2.   Diffusion:   Ensures that changing one bit of plaintext affects many ciphertext bits.  o   Achieved using   permutation   (rearranging bits in the block).
3.   Key Expansion:   Generates multiple subkeys from the main key for multiple rounds.  Example:  In   DES , confusion is done using   S-boxes , and diffusion is done through  permutations .  3. Block Cipher Modes of Operation  Since block ciphers encrypt fixed-size blocks, different   modes of operation   are used to securely encrypt data longer than the block size.  Common Modes of Operation:  1.  Electronic Codebook (ECB)  2.  o   Each block is encrypted separately.  o   Weakness:   Identical plaintext blocks produce identical ciphertext blocks (vulnerable to pattern attacks).  o   ❌   Not recommended for security . 3.  Cipher Block Chaining (CBC)  4.  o   Uses an   Initialization Vector (IV)   to XOR with the first plaintext block before encryption.  o   Ensures different ciphertexts for the same plaintext.  o   ✅   More secure than ECB . 5.  Counter (CTR) Mode  6.  o   Converts a block cipher into a   stream cipher   by using a counter that is encrypted and XORed with the plaintext.  o   ✅   Fast and supports parallel encryption . 7.  Galois/Counter Mode (GCM)
8.  o   A   modified CTR mode   with built-in authentication.  o   ✅   Used in modern encryption standards .  4. Advanced Encryption Standard (AES)  AES is the most widely used modern encryption standard, developed as a replacement for DES.  Key Features of AES:     Block Size:   128 bits.     Key Sizes:   128, 192, or 256 bits.     Rounds:  o   10 rounds for   128-bit keys .  o   12 rounds for   192-bit keys .  o   14 rounds for   256-bit keys .     Structure:   Uses the   Substitution-Permutation Network (SPN)   instead of a Feistel network.  Main Operations in AES:  1.   SubBytes:   Uses an   S-box   to replace bytes for confusion. 2.   ShiftRows:   Shifts rows in the state matrix for diffusion. 3.   MixColumns:   Performs a linear transformation for further diffusion. 4.   AddRoundKey:   XORs the data with a subkey.  ✅   Used in:   Secure communication (SSL/TLS), disk encryption, Wi-Fi (WPA2), and more.  5. Triple DES (3DES)  To improve the security of DES,   Triple DES (3DES)   was developed.  How 3DES Works:     Encrypts data   three times   using DES with two or three keys.     Key Length:   112-bit or 168-bit (when using 3 different keys).     More secure than DES , but slower than AES.  ❌   Disadvantage:     3DES is now being   phased out   because AES is faster and more secure.
✅   Used in:   Older banking systems.  6. Blowfish Algorithm  Blowfish is a   fast and secure   block cipher designed by Bruce Schneier in 1993.  Key Features of Blowfish:     Block Size:   64 bits.     Key Length:   32–448 bits.     Rounds:   16 rounds.     Structure:   Uses a   Feistel network   like DES.  ✅   Advantages:     Faster than DES and 3DES .     Flexible key size   allows strong encryption.  ❌   Disadvantage:     The   64-bit block size   makes it vulnerable to birthday attacks on large datasets.  ✅   Used in:   Secure storage, password hashing (bcrypt).  7. RC5 Algorithm  RC5 (Rivest Cipher 5) is a   lightweight   and   configurable   encryption algorithm developed by Ronald Rivest.  Key Features of RC5:     Block Size:   32, 64, or 128 bits.     Key Length:   0 to 2048 bits (flexible).     Rounds:   Variable (typically 12 or more).     Structure:   Uses   data-dependent rotations , making it unique.  ✅   Advantages:     Simple and   fast encryption .     Supports   different key and block sizes .  ✅   Used in:   VPNs, cryptographic protocols.

While DES and 3DES are outdated,   AES is the modern standard   due to its security and efficiency. Other algorithms like Blowfish and RC5 are also used in specific applications.  Public Key Cryptography & Key Exchange Methods  Public Key Cryptography (also known as Asymmetric Cryptography) is a cryptographic system that uses   two different keys —one for encryption and one for decryption. It solves the major limitation of symmetric encryption:   secure key exchange .  Key Features:     Uses a   public key   for encryption and a   private key   for decryption.     Provides   confidentiality, authentication, and integrity .     Used in   digital signatures, secure web communication (SSL/TLS), and cryptocurrency .  1. Principles of Public Key Cryptosystems  How it Works:  1.   The sender   encrypts   the message using the   receiver’s public key . 2.   The receiver   decrypts   the message using their   private key .
3.   Only the   private key   can decrypt the message, ensuring   security .  Key Applications:  ✔   Secure communication (e.g., HTTPS, SSL/TLS).  ✔   Digital signatures and authentication.  ✔   Cryptographic key exchange.  2. RSA Algorithm (Rivest-Shamir-Adleman)  RSA is one of the most widely used public key cryptosystems. It relies on the difficulty of   factoring large prime numbers .  Steps of RSA Encryption:  1.  Key Generation:  2.  o   Choose two large prime numbers   p   and   q .  o   Compute   n = p × q   (this is the   modulus ).  o   Compute   ϕ(n) = (p - 1) × (q - 1) .  o   Choose a public exponent   e   (commonly   65537 ).  o   Compute the private key   d   using   d ≡ e⁻¹ (mod ϕ(n)) . 3.  Encryption:  4.  o   Convert plaintext   M   into a number.  o   Compute ciphertext   C = M^e mod n . 5.  Decryption:  6.  o   Compute plaintext   M = C^d mod n .  ✅   Strength:   Security is based on   factoring large numbers , which is computationally difficult.  ❌   Weakness:   Slow for large-scale data encryption (often combined with symmetric encryption like AES).
✔   Used in:   SSL/TLS, digital signatures, banking security.  3. Key Management  Why is Key Management Important?     Public key cryptography requires   secure distribution, storage, and revocation   of keys.     Improper management can   expose private keys   and compromise security.  Common Key Management Methods:  1.  Public Key Infrastructure (PKI):  2.  o   Uses   digital certificates   issued by a   Certificate Authority (CA) .  o   Ensures   trust   between users and services. 3.  Key Revocation and Renewal:  4.  o   If a key is   compromised , it must be revoked and replaced. 5.  Hardware Security Modules (HSMs):  6.  o   Physical devices that   store and protect   cryptographic keys.  ✔   Used in:   Secure authentication, digital certificates, cloud security.  4. Diffie-Hellman Key Exchange  Diffie-Hellman (DH) is a method for   securely exchanging cryptographic keys   over an insecure channel.  How it Works:
1.   Two users agree on a   large prime number (p)   and a   generator (g) . 2.   Each user picks a   private key (a & b)   and computes their   public key :  o   Alice sends   A = g^a mod p .  o   Bob sends   B = g^b mod p . 3.   Both users compute the   shared secret :  o   Alice computes   S = B^a mod p .  o   Bob computes   S = A^b mod p .  o   Since   S = g^(ab) mod p , both get the same key.  ✅   Strength:   No secret key is ever transmitted, making it secure against eavesdropping.  ❌   Weakness:   Vulnerable to   man-in-the-middle attacks   (unless combined with authentication).  ✔   Used in:   Secure network protocols (SSH, SSL/TLS, VPNs).  5. Elliptic Curve Arithmetic  Elliptic Curve Cryptography (ECC) is an alternative to RSA, offering the   same security with smaller key sizes . It is based on   Elliptic Curve Mathematics .  Elliptic Curve Equation:  A general elliptic curve equation is:  y2=x3+ax+by^2 = x^3 + ax + by2=x3+ax+b  where   a   and   b   define the curve.  Elliptic Curve Operations:  1.   Point Addition:   Adding two points on the curve results in another point. 2.   Point Doubling:   Adding a point to itself. 3.   Scalar Multiplication:   Multiplying a point by a large number.  Example:   If   P   is a point on the curve, then  Q=kPQ = kPQ=kP  (where   k   is a private key) is easy to compute, but   recovering k   from Q (discrete logarithm problem) is very hard.  ✅   Advantage:   Provides   strong security with smaller key sizes   (e.g.,   256-bit ECC = 3072-bit RSA ).  ✔   Used in:   Bitcoin, TLS, secure messaging (Signal, WhatsApp).
6. Elliptic Curve Cryptography (ECC)  ECC is an advanced cryptographic method using elliptic curve arithmetic for   secure encryption and key exchange .  Why ECC is Better than RSA?     Faster and more efficient   for encryption/decryption.     Smaller key sizes   →   Lower storage and processing power needed .     More resistant to   quantum computing attacks .  Common ECC Algorithms:  1.  Elliptic Curve Diffie-Hellman (ECDH):  2.  o   Used for   secure key exchange .  o   Similar to Diffie-Hellman but uses elliptic curve mathematics. 3.  Elliptic Curve Digital Signature Algorithm (ECDSA):  4.  o   Used for   digital signatures .  o   Faster than RSA signatures.  ✅   Used in:   Cryptocurrency (Bitcoin, Ethereum), secure websites (TLS/SSL), IoT security.
